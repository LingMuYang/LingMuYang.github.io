项目部署(入门篇)

本篇播客讲的是如何在linux上如何部署一个新完成的项目，如有错误请联系15076117282@163.com

Java环境

0.下载jdk8

去官网下载最新安装包(可在Windows下下载完成后，通过文件夹共享到Linux上)

1.登录Linux，切换到root用户

su root获取root用户权限，当前工作目录不变（需要root密码）
或 sudo -i不需要root密码直接切换成root（需要当前用户密码）

2.在usr目录下建立java安装目录

cd /usr
mkdir java

3.jdk-8u60-linux-x64.tar.gz拷贝到java目录下

cp/mnt/hgfs/linux/jdk-8u60-linux-x64.tar.gz/usr/java/
（版本根据自己需要自行选择）

4.解压jdk到当前目录

tar -zxvf jdk-8u60-linux-x64.tar.gz
得到文件夹jdk1.8.0_60

5.安装完毕为他建立一个链接以节省目录长度

In -s/usr/java/jdk1.8.0_60/ /usr/jdk

6.编辑配置文件，配置环境变量

vim /etc/profile

添加如下内容：JAVA_HOME根据实际目录来
JAVA_HOME=/usr/java/jdk1.8.0_60
CLASSPATH=$JAVA_HOME/lib/
PATH=$PATH:$JAVA_HOME/bin
export PATH JAVA_HOME CLASSPATH

7.重启机器或者执行命令

source /etc/profile

8.查看安装情况

java -version

###Tomcat安装开始
1.先在/usr下新建tomcat文件夹，并把安装包拷贝进来
cd /usr/tomcat
tar xvf apache-tomcat-8.5.12.tar.gz
2.配置环境变量 vi /etc/profile
CATALINA=/usr/tomcat/apache-tomcat-8.5.12
3.使修改生效
source /etc/profile
4.启动tomcat
在bin目录下./startup.sh

myeclipse

本片博客记录一些自己在使用myeclipse中遇到的一些问题，如有错误请联系
15076117282@163.com

1.新安装MyEclipse，导入以前做的程序后程序里好多错误，提示the import java.util cannot be resolved

原因：

这是由于你的项目buildpath不对，原来的项目，比如采用了原先的MyEclipse自带的jdk（D:\myeclipse\xxxx）,结果，你现在换了一个，原来的没了，就导致出现这种错误，基本的java.lang或者util包都找不到

解决方案

右键项目—》buildpath—》最下边那个confiura…的选择libraies
找到JRE（这个时候你会发现这个jre前面有!或者是红x）
选择remove掉
重新为该项目选择一个JRE
选中项目，project—-clean
刷新项目，问题就解决了
